# Product

## Register

product

## Users

The primary user is one individual lifter using a phone in the gym to run repeatable Workout A / Workout B routines. The user already knows the exercises and needs fast access to setup notes, target sets and reps, current weight, previous-result guidance, rest timing, and current result marking while physically tired and mid-session.

## Product Purpose

Fitness Hub is a personal gym-session command panel for executing known workouts with minimal friction. It exists to keep the current workout moving: start or resume a session, scan the next exercise, adjust weight/setup/targets when needed, mark success or failure, use the rest timer, and review local history. Success means the app stays out of the way, preserves context, and never navigates away from the current workout unless the user explicitly taps navigation.

## Brand Personality

Fast, focused, no-nonsense, compact, calm, low-friction, phone-first, and utility-minded. The interface should feel like a sharp tool built for repeated use under physical fatigue: direct labels, dense but readable information, clear state feedback, and no ornamental ceremony.

## Anti-references

This should not look or behave like a social fitness app, habit app, gamified wellness app, generic SaaS dashboard, RPG tracker, or workout marketplace. Avoid gamer colors, badges, achievements, childish emojis, excessive gradients or glows, wellness-app gloss, huge hero sections, repeated information, nested card clutter, and decorative UI that slows scanning.

## Design Principles

- Keep the workout in hand: preserve session context, make navigation explicit, and avoid any surprise route changes.
- Optimize for tired scanning: prioritize legible hierarchy, compact groupings, visible state, and large enough touch targets over decoration.
- Show only useful controls: keep keyboard input rare and manual, reserving it for edits such as weight, setup, and target changes.
- Make status impossible to miss: current result, previous guidance, completion progress, and timer feedback should be visually distinct without becoming loud.
- Behave like a utility: consistent components, predictable gestures, restrained motion, and no social or game mechanics.

## Accessibility & Inclusion

Target high contrast, readable phone text, touch-friendly controls, clear selected states, visible timer and status feedback, and reduced-motion support where practical. The app should remain usable one-handed in a gym setting, with no keyboard appearing unless the user intentionally edits a field.
